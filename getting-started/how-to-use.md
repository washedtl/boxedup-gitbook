# How to use this guide

This guide is sequenced. If you're new, **read in order** — every section assumes you've internalized what came before.

## The three reading paths

### 🟢 Beginner path (haven't sold anything yet)

Read straight through Parts 1 → 2 → 3 → 4 → 6. Skip Part 5 (BoxedUp CC) until you've made your first 10 flips manually — you need to feel the pain of doing it by hand before tools save you time.

**Why this order:** Account → FBA mechanics → sourcing methods → tools → deal analysis. Each one unlocks the next.

### 🟡 Intermediate path (selling, want to scale)

You probably already have an account and have done some RA. Skim Part 1, read Part 2 carefully (most beginners skip the fee math and bleed money), then go deep on:

- [Building a sourcing routine that scales](../sourcing/sourcing-routine.md)
- [Tools workflow](../tools/keepa.md) — especially Keepa
- [Using BoxedUp Command Center](../boxedup-cc/overview.md)
- [Deal analysis](../deal-analysis/profit-equation.md)

### 🔴 Advanced path (hiring, building infra, scaling 5-figure+ months)

Use the sidebar. The high-leverage sections for you:

- [Reimbursements](../seller-central/reimbursements.md) — most resellers leave \$500–\$2K/mo on the table
- [Account health & survival](../anti-patterns/suspension.md)
- [Bookkeeping](../business-setup/bookkeeping.md)
- [Anti-patterns](../anti-patterns/ip-complaints.md)

## Conventions used

| Symbol | Meaning |
|---|---|
| 🟢 / 🟡 / 🔴 | Difficulty: beginner / intermediate / advanced |
| 💰 | Fee math or money-flow detail |
| ⚠️ | Risk, gotcha, or "this will get you suspended" |
| 🛠️ | Tool walkthrough |
| 📊 | Real numbers from BoxedUp Discord flips |

> **Hint blocks like this** are tactical tips. Don't skip them.

⚠️ **Warning blocks like this** mean someone in the Discord lost money learning this the hard way.

## Updating the guide

Found something stale, wrong, or missing? Two ways to flag it:

1. Drop a message in `#guide-feedback` on Discord with the page name.
2. If you're a contributor, edit the page in GitBook directly.

The bottom of every page has a **last updated** date. If a page is older than 6 months and references prices/fees, double-check the current numbers in Seller Central before acting.
